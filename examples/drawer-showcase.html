Drawer Components Examples

This page demonstrates the usage of two web components for implementing drawer UI patterns:

  • dry-drawer - A basic drawer component that slides in from either side
  • ajax-drawer - An extended drawer that loads content via AJAX using HTMX

Basic Drawer (dry-drawer)

Right Side Drawer (Default)

Right Drawer Example

This is a basic drawer that slides in from the right side of the screen.

Features:

  • Custom styling via attributes
  • Backdrop click to close
  • ESC key to close
  • Smooth animations

Left Side Drawer

Left Drawer Example

This drawer slides in from the left side of the screen.

You can customize:

  • Width
  • Background color
  • Header content
  • Trigger button style
HTML 
<dry-drawer
    position="right"
    drawer-class="bg-white w-80"
    button-class="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded"
    trigger-content="Open Right Drawer">
    <span slot="header">Right Drawer Example</span>
    <div slot="content">
        <p>This is a basic drawer that slides in from the right side.</p>
        <!-- More content here -->
    </div>
</dry-drawer>

API Reference

Common Attributes

Attribute Description Default
position Drawer position, either "left" or "right" "right"
drawer-class CSS classes for the drawer element "bg-white w-80"
header-class CSS classes for the header ""
backdrop-class CSS classes for the backdrop ""
button-class CSS classes for the trigger button "bg-blue-500 text-white px-4 py-2 rounded"
trigger-content Text for the trigger button "Open Drawer" / "Load Content"
header-content Text for the drawer header (alternative to slot) "Drawer"

AJAX Drawer Specific Attributes

Attribute Description Default
url URL to fetch content from ""
trigger-type HTMX trigger type "click"
content-id ID for the content container (should be unique) Auto-generated random ID

Events

Both drawer components emit the following custom events:

Event Description
drawer:opened Fired when the drawer is opened
drawer:closed Fired when the drawer is closed

Slots

Both drawer components support the following slots for content customization:

Slot Description
header Content for the drawer header
content Main content for the drawer (only used in dry-drawer)

HTMX Integration Notes

The AJAX drawer component integrates with HTMX for content loading. Here are some key features:

  • Content is loaded using HTMX's hx-get attribute
  • You can control when the request is triggered with the trigger-type attribute
  • The drawer automatically opens when the request starts
  • Loading and error states are handled automatically
  • You can close the drawer programmatically by returning the HX-CloseDrawer: close header in your response
JavaScript 
// Server-side example (Node.js/Express)
app.get('/api/drawer-content.html', (req, res) => {
  // Do some processing...

  // Optionally close the drawer after processing
  res.setHeader('HX-CloseDrawer', 'close');

  // Send the response
  res.send('<div>This content was loaded via AJAX</div>');
});

JavaScript API

You can also control the drawers programmatically:

Programmatic Control

JavaScript Controlled Drawer

This drawer is controlled via JavaScript.
JavaScript 
// Get a reference to the drawer
const drawer = document.getElementById('js-controlled-drawer');

// Open the drawer
drawer.open();

// Close the drawer
drawer.close();

// Listen for events
drawer.addEventListener('drawer:opened', () => {
  console.log('Drawer was opened');
});

drawer.addEventListener('drawer:closed', () => {
  console.log('Drawer was closed');
});
